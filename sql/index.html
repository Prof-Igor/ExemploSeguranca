Vulnerabilidades de SQL Injection

Exemplo 1: Autenticação de Usuário

Quando um aplicativo permite que os usuários insiram seu nome de usuário e senha, se não forem feitos de forma segura, um invasor pode tentar usar uma injeção SQL como:

admin' --

Tente inserir a entrada acima para "Nome de Usuário" para ver a vulnerabilidade.

Exemplo 2: Pesquisa de Produtos

Um aplicativo que permite que os usuários pesquisem produtos pode ter a seguinte consulta SQL vulnerável:

SELECT * FROM produtos WHERE nome = '$nome'

Um atacante pode usar a entrada:

produto'; DROP TABLE produtos; --

Tente inserir a entrada acima para "Nome do Produto" para ver a vulnerabilidade.

Exemplo 3: Exibição de Detalhes do Usuário

Um aplicativo que exibe detalhes do usuário pode usar uma consulta SQL como:

SELECT * FROM usuarios WHERE id = '$id'

Um invasor pode tentar:

1 OR 1=1

Tente inserir a entrada acima para "ID do Usuário" para ver a vulnerabilidade.
